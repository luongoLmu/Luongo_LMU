Graph Database

- A graph database is a type of database that uses graph structures with nodes, edges, and properties to represent and store data
- In a graph database it shows data and the relationships between data points kind of like a mind mapp
- This makes graph databases particularly well-suited for scenarios where relationships between entities play a crucial role,
such as in social networks, fraud detection, network analysis
- Graph databases offer advantages in scenarios where the relationships between
entities are complex and need to be traversed efficiently

Key Components:

Node: A node represents an entity in the graph database. Nodes can have properties that store information about the entity.
Edge: An edge represents a relationship between two nodes. Edges can also have properties to describe the nature of the relationship.
Property: Properties are key-value pairs associated with nodes and edges, providing additional information about them.
